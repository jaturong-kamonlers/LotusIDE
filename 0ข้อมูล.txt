  ---
  ความท้าทายหลัก

  1. Compilation Pipeline

  ปัญหาที่ยากที่สุด — ต้องเรียก compiler (avr-gcc, arm-gcc, bossac ฯลฯ)

  ┌───────────────────────────────────┬─────────┬───────┬─────┬─────────┐
  │                วิธี                 │ Windows │ Linux │ ARM │ Browser │
  ├───────────────────────────────────┼─────────┼───────┼─────┼─────────┤
  │ Bundle compiler ไว้เอง (แบบเดิม)    │ ✓       │ ✓     │ ✓   │ ✗       │
  ├───────────────────────────────────┼─────────┼───────┼─────┼─────────┤
  │ arduino-cli (แนะนำ)               │ ✓       │ ✓     │ ✓   │ ✓*      │
  ├───────────────────────────────────┼─────────┼───────┼─────┼─────────┤
  │ Cloud compile (เช่น Arduino Cloud) │ ✓       │ ✓     │ ✓   │ ✓       │
  └───────────────────────────────────┴─────────┴───────┴─────┴─────────┘

  ▎ *arduino-cli มี daemon mode (HTTP server) — browser เรียก compile ผ่าน local server ได้

  2. Serial Port

  ┌─────────────────────────────┬─────────┬───────┬─────┬─────────┐
  │             API             │ Windows │ Linux │ ARM │ Browser │
  ├─────────────────────────────┼─────────┼───────┼─────┼─────────┤
  │ @serialport/serialport      │ ✓       │ ✓     │ ✓   │ ✗       │
  ├─────────────────────────────┼─────────┼───────┼─────┼─────────┤
  │ Web Serial API (Chrome 89+) │ ✓       │ ✓     │ ✓   │ ✓       │
  └─────────────────────────────┴─────────┴───────┴─────┴─────────┘

  Web Serial API ทำงานได้ทั้ง Electron และ Browser Chrome — นี่คือ key ที่ทำให้ทำ browser version ได้

  ---
  Tech Stack ที่แนะนำ

  Desktop (Windows/Linux/ARM):
    Electron + Vue 3 + Vite + Vuetify 3

  Browser (อนาคต):
    เดิม Vue 3 app เดียวกัน → ใส่ใน PWA
    compile ผ่าน arduino-cli daemon หรือ cloud

  เหตุผลที่เลือก Vue 3 + Vite แทน Vue 2 (แบบเดิม)

  - Build เร็วกว่า 10-20x
  - Composition API สะอาดกว่า
  - รองรับ browser/PWA ได้ดีกว่า
  - Vuetify 3 รองรับดีกว่า

  ---
  Architecture ที่ออกแบบไว้

  C:\LotusIDE\
  ├── electron/              ← Electron main process
  ├── src/                   ← Vue 3 app (web-first design)
  │   ├── core/              ← business logic (platform-agnostic)
  │   │   ├── compiler.js    ← เรียก arduino-cli
  │   │   ├── serial.js      ← Web Serial API (ทำงานได้ทุกที่)
  │   │   └── boards.js      ← board management
  │   ├── blockly/           ← blocks + generators (ย้ายจากเดิมได้เลย)
  │   ├── boards/            ← LotusDevkit, LotusDevkit4Wheels (ของคุณ)
  │   └── components/        ← Vue components
  ├── arduino-cli/           ← arduino-cli binary (per platform)
  └── package.json
